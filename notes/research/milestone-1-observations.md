# Milestone 1 Observations

- Launcher probe status: `captures/20260418-154355-runner/launcher-http.log` recorded `GET /rss` and `GET /updatev2/MiniUpdate/update.info` from the stock launcher. `HttpProbeService` answered with `SERVER ONLINE!` and `NO_UPDATE`, and the launcher proceeded to start `Game.exe`.
- Server list status: decoded the stock `SvrList.dta` and `SvrListM.dta` formats into structured rows with `emulator.tools.server_list`. The classic login entry resolves to `GLOBAL 79.137.101.33 4005`, and the menu server list rewrites cleanly to a localhost endpoint while preserving the original row shape.
- Login gateway status: the first localhost-captured login packet is `captures/20260418-162948-runner/packets/conn-0001-in-0001.bin` at 74 bytes. Its first 16 bytes are `1100b10b7a975ec71045e61875b433ab`, and the exact request was saved to `notes/research/login-script.json`.
- Scripted replay status: the initial 4-byte stub reply `90000000` was implemented and replayed successfully at the transport layer, but the client did not advance with that minimal response alone. The real remote failure reply was then captured as `notes/research/login-failure-script.json`, and the replay engine gained `close_after_response` support so failure scripts can mimic the upstream disconnect behavior exactly.
- Proxy gateway status: `LoginGatewayService` now supports upstream proxy mode in addition to raw capture and scripted replay. The successful live capture run used `127.0.0.1:14021 -> 79.137.101.33:4005` while preserving every inbound and outbound packet locally under `captures/20260418-162948-runner`.
- Live auth outcome: the proxied localhost login path now works end-to-end. The user confirmed that they were able to log in and create a character from the stock client against the localhost shim.
- Packet notes: the successful proxied run expanded to 15 inbound packets and 7 outbound packets. Notable replies include `conn-0001-out-0002.bin` at 50 bytes, `conn-0001-out-0004.bin` at 28 bytes with an ASCII fragment containing `check 0`, `conn-0001-out-0006.bin` at 182 bytes, and `conn-0001-out-0007.bin` at 2737 bytes. Plaintext fragments observed in proxied server traffic include `art`, `33114`, `check 0`, and `OK`.
- Client startup pitfall: the `Unable to read skill information.` popup was not a server failure. Ghidra traced it to `FUN_0076d990`, which opens relative files such as `data/menu/str/US.txt`; launching `Game.exe` without the client directory as the working directory causes that read to fail even when the file exists.
- Extra service forwarders: ad-hoc localhost proxies for ports `4007` and `4008` were also run and logged under `captures/20260418-165923-proxy-4007` and `captures/20260418-165923-proxy-4008` so later milestones can inspect post-login service traffic separately from the main login gateway.
